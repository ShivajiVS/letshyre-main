1. PROJECT OVERVIEW

Type:

Recruitment/Job Matching Platform - Frontend Application

4. KEY FEATURES & WORKFLOWS

A. Authentication System

Login Flow:

  1. Candidate enters email + password
  2. Service sends credentials to backend /user/v1/login/
  3. Backend returns access token + refresh token + user data
  4. Tokens stored in localStorage
  5. User redirected to dashboard

Token Management:

  • Access token attached to all authenticated API requests
  • If 401 error received → automatically refresh token
  • If refresh fails → logout and redirect to login

Public Endpoints (no token required):

  • OTP sending/verification
  • User registration
  • Email verification

B. Registration System

Multi-Step Process:

  1. Email - Candidate provides name, username, email
  2. Email OTP - Verification code sent & confirmed
  3. Mobile - Candidate provides phone number
  4. Mobile OTP - Verification code sent & confirmed
  5. Password - Set secure password
  6. Success - Account created

C. Candidate Dashboard

After Login:

  1. Checks if profile is completed (localStorage flag)
  2. If not completed → shows profile stepper overlay
  3. If completed → shows main dashboard

Dashboard Sections (Tab-based):

  • Find Jobs - Browse & search available jobs with carousel and filters
  • My Applications - Jobs applied to (locked until profile complete)
  • Scorecard - Performance/rating metrics (locked until profile complete)

D. Profile Management

4-Step Profile Completion:

  1. Identity - Basic info (name, email, phone)
  2. Role - Career preferences & skills
  3. Resume - Upload/attach resume
  4. Review - Confirm all information before submission

6. DATA FLOW ARCHITECTURE

Request & Response Flow:

User Interface (Components) ↓ Services Layer (API calls) ↓ Axios HTTP Client ↓ Backend API ↓ Axios Response Interceptor ↓ Store in localStorage / State ↓ Update React Components

Authentication Token Flow:

Login → Store Token in localStorage ↓ API Request → Add token to header ↓ 401 Response → Attempt refresh ↓ New Token → Retry original request ↓ Refresh fails → Logout & Redirect to login
